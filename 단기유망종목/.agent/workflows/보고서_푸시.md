# 🚀 단기 유망 종목 보고서 웹페이지 푸시 (보고서 푸시)

이 워크플로우는 사용자가 **"push 하자"**라고 요청할 때 시작되며, 현재 폴더(`유망종목-단기`)에서 생성된 최신 단기 유망 종목 보고서를 웹사이트(`krx-intelligence`)의 유망종목 메뉴(`content/picks`)로 업데이트하고 GitHub에 반영(push)합니다.

## 📋 수행 단계 (Operation Steps)

### Step 1. 최신 보고서 식별 및 준비
- 현재 폴더에서 `YYYYMMDD_단기유망종목_발굴리포트.md` 형식의 가장 최신 파일을 식별합니다. (예: `20260318-genesis-report.mdx` 포맷으로 변환 예정)

### Step 2. 웹 프로젝트 대상 폴더로 복사
- **대상 폴더**: `c:\Users\hwang\OneDrive\바탕 화면\안티그라비티폴더\주식 정보 웹페이지\krx-intelligence\content\picks`
- **파일명 변경**: `YYYYMMDD-genesis-report.mdx` (예: `20260318-genesis-report.mdx`)
- **확장자 변환**: `.md` -> `.mdx`로 변환하여 복사합니다.

### Step 3. MDX 프론트매터(Frontmatter) 엄격 검증
- **[경고]** **ticker, market, term, currentPrice, targetPrice, stopLoss** 필드는 웹사이트 노출을 위한 **필수 메타데이터**입니다. 하나라도 누락될 경우 배포 후 웹페이지에 리포트가 나타나지 않습니다.
- **필수 포함 필드 (StockPick Schema)**:
    1.  `title`: 리포트 제목 (예: "2026년 03월 27일 단기 유망 종목 (제네시스 리포트)")
    2.  `ticker`: "GENESIS" (종합 리포트의 경우)
    3.  `market`: "KOSDAQ" (또는 KOSPI)
    4.  `term`: "short"
    5.  `date`: "YYYY-MM-DD" (반드시 따옴표 포함)
    6.  `currentPrice`: 0 (종합 리포트의 경우)
    7.  `targetPrice`: 0 (종합 리포트의 경우)
    8.  `stopLoss`: 0 (종합 리포트의 경우)
    9.  `summary`: 핵심 요약 문구
    10. `category`: (제거) 스키마에 정의되지 않은 필드 사용 금지
    11. `status`: "active"
    12. `tags`: ["단기유망", "KOSPI", "KOSDAQ", "주요섹터"]
- **체크리스트**:
    - `ticker`, `market`, `term` 등의 필드가 누락되지 않았는가?
    - `Price`와 `Loss` 필드가 숫자로 입력되었는가? (0 이라도 포함 필요)
    - `date` 형식이 따옴표와 함께 일치하는가?
- **포맷 오류 발생 시**: `Contentlayer`가 문서를 스킵하여 웹사이트에 노출되지 않거나 빌드 에러가 발생합니다. 반드시 최종 확인 후 푸시합니다.

### Step 4. GitHub 저장소 푸시 및 동기화
- **작업 경로**: `c:\Users\hwang\OneDrive\바탕 화면\안티그라비티폴더\주식 정보 웹페이지\krx-intelligence`

- **커맨드 수행 (원스톱 처리)**: 
    1.  `git add content/picks/YYYYMMDD-genesis-report.mdx`
    2.  `git commit -m "feat: [YYYY-MM-DD] 단기 유망 종목 보고서 업데이트"`
    3.  `git pull --rebase origin main` (원격 저장소와의 충돌 방지 및 최신화)
    4.  `git push origin main`

### Step 5. 완료 확인 및 캐시 초기화

- 필요 시 빌드 로그를 간략히 체크하여 누락 필드 유무를 최종 확인합니다.

## 🚦 운영 원칙
1.  **파일명 준수**: `YYYYMMDD-genesis-report.mdx` 형식을 엄격히 지켜 시스템 인덱싱 오류를 방지합니다.
2.  **프론트매터 무결성**: **[필수]** 배포 실패를 방지하기 위해 MDX 프론트매터 필드와 데이터 포맷을 엄격하게 검증합니다. (특히 `date` 필드 주의)
3.  **데이터 무결성**: 복사 및 변환 과정에서 원본 분석 수치가 변동되지 않도록 주의합니다.
4.  **푸시 전 확인**: 파일이 웹 프로젝트 폴더에 정확히 복사되었는지 최종 확인 후 git 명령어를 실행합니다.
5.  **스키마 완결성**: **[최종 점검]** PUSH 직전, `ticker`, `market`, `term`, `currentPrice`, `targetPrice`, `stopLoss` 필드가 모두 포함되었는지 다시 한번 확인하여 웹페이지 노출 오류를 차단합니다.
