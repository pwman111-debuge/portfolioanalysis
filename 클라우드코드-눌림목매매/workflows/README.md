# 유망종목 & 대세섹터 판별 워크플로우

> **목적**: 기존 보유 포트폴리오와 **독립적으로**, 시장에서 단기·중기·장기 유망종목을 신규 발굴하고 **대세(+)/대세(−)** 속성을 태깅한다.
> **출력**: 6개 버킷 × N종목 리스트 — 단기×대세(+)/대세(−), 중기×대세(+)/대세(−), 장기×대세(+)/대세(−)
> **제외**: 보유 종목 리뷰, 분할 매수 2차 타점, 포트 리밸런싱 — 별도 워크플로우(`포트폴리오관리/.agents/`)에서 처리

## 🔍 판별 기준 요약 비교

| 항목 | 단기 | 중기 | 장기 |
|------|------|------|------|
| **보유 기간** | 5~10거래일 (1~2주, 최대 3주) | 3~6개월 | 1~3년 |
| **선정 개수** | 3종목 (방어 시 2) | 3종목 | 3종목 |
| **핵심 필터** | 수급 + 기술적 타점 | 펀더+수급 스코어 14점↑ | ROE 15%+, Moat |
| **시가총액** | - | 500억↑ | 500억↑ |
| **영업이익 조건** | - | 최근 분기 흑자, YoY +10%↑ | 3년 연속 우상향 |
| **재무 조건** | - | ROE 10%↑, 부채 150%↓ | ROE 15%↑, 부채 200%↓ |
| **밸류에이션** | - | PER 섹터 할인 or PEG<1 | PER 섹터比 0.5~1.2, PBR≤2.0 |
| **기술적 조건** | 5/20/60일 정배열 | 20일·60일선 위 | 52주 저점 대비 30% 이내 |
| **R:R 기준** | 2:1 이상 필수 | 목표-손절 명시 | 12/24/36개월 목표가 제시 |
| **치명 리스크** | 5일 내 유증/CB 공시 | 권리변경·단기 과열 | 부정 이벤트·경영진 교체 |

## 🎯 대세섹터 분류 요약

### 편입 (대세)
```
고정: AI인프라 + 전력설비 + 원전/SMR + 방산
동적: 시황분석의 "주도 섹터" 추가 편입
```

### 결과 — 익절/손절 기준 완화 (수익 극대화)
| | 중기×대세 | 장기×대세 |
|---|---|---|
| 1차 익절 | +20% | +50% |
| 2차 손절 | -15% | -25% |

### 강등 (대세 → 일반)
```
🔻 글로벌 선도 기업 실적 쇼크
🔻 핵심 정책 폐기 / 예산 삭감
🔻 섹터 전체 외인·기관 3주 연속 이탈
🔻 대체 기술 등장 → thesis 훼손
```

## 🧭 사용 흐름 (신규 발굴 전용) — "제네시스 하자" 트리거

```
[0] 개념 숙지 (00_눌림목-정의.md)
     ↓
[1] 시황 판단           ← naver_finance.py market
     ↓                   → 공격 / 중립 / 방어
[2] 대세섹터 확정       ← naver_finance.py sector / theme
     ↓                   → 고정 4대 + 당일 주도 섹터
[3] 기간별 후보 풀 수집 (단·중·장 병렬)
     ├── 단기 풀 5~7    ← 단기 수급 스크리닝
     ├── 중기 풀 5~7    ← naver_finance.py screen / all / investor / short
     └── 장기 풀 5~8    ← gene-scan.py --sector / --tickers --detail
     ↓
[4] 눌림목 건강도 스코어링 (일괄 실행)
     └── pullback_score.py <codes...> --json
     ↓                   → 0~100점, 배제 종목 자동 탈락
[5] 대세(+)/대세(−) 태깅 → 6버킷 분류
     ↓
[6] 최종 리포트 생성
     └── workflows/reports/YYYYMMDD_발굴리포트.md
         → 건강한 눌림목 순 (점수 DESC → 20일선 거리 ASC → 시총 DESC)
```

> ⚠️ 이 워크플로우의 출력은 **후보 리스트**까지. 실제 매수/보유 관리는 `포트폴리오관리/` 폴더의 별도 워크플로우.

## ⚙ 공통 운영 원칙

1. **발굴 전용** — 이 워크플로우는 신규 유망 종목을 찾아 6버킷으로 태깅할 뿐, 매수·보유 관리는 포함하지 않음
2. **시황 판단이 선결 조건** — 방어 시황 시 발굴 자체는 수행하되 "진입 보류" 태그 부착
3. **눌림목 타점은 필터이지 필수 조건 아님** — 눌림 타점 미도달 종목은 "관망 대기"로 별도 태깅
4. **R:R 및 가격 전략 필수** — 현재가·목표가·1차/2차 손절가 모든 버킷 공통
5. **데이터 출처: 네이버 증권 우선** (추정값·기억값 금지)
6. **최종 결정권은 황원장** — Claude는 기준에 따른 후보 제안만 수행

## 📦 출력 버킷 정의

| 버킷 | 기간 | 섹터 | 익절·손절 기준 |
|------|------|------|----------------|
| 단기 × 대세(+) | 1~2주 | 대세섹터 해당 | R:R 2:1 (기본) |
| 단기 × 대세(−) | 1~2주 | 일반섹터 | R:R 2:1 (기본) |
| 중기 × 대세(+) | 3~6개월 | 대세섹터 해당 | +20%/+40%/+60%, 손절 -15% |
| 중기 × 대세(−) | 3~6개월 | 일반섹터 | +15%/+25%/목표가, 손절 -12% |
| 장기 × 대세(+) | 1~3년 | 대세섹터 해당 | +50%/+100%/+150%, 손절 -25% |
| 장기 × 대세(−) | 1~3년 | 일반섹터 | +30%/+50%/thesis, 손절 -20% |

> 익절·손절 기준은 발굴 결과에 참고용으로 표기. 실제 집행은 포트폴리오 워크플로우에서 적용.
