# 발굴 실행 워크플로우 (Discovery Execution)

> **역할**: `00_눌림목-정의.md` ~ `04_대세섹터-판별기준.md`를 오케스트레이션하여 **단·중·장 × 대세(+)/대세(−) 6버킷**에 유망 종목을 채우고, **눌림목 건강도 순으로 정렬된 리포트**를 생성한다.
>
> **입력**: 없음 (당일 시황·지수 자동 수집)
> **출력**: `workflows/reports/YYYYMMDD_발굴리포트.md` (단·중·장 섹션 + 건강도 순 종목 리스트)

---

## 실행 트리거

사용자가 **"제네시스 하자"** 또는 **"/discover"** 입력 시 이 워크플로우 시작.

---

## 전체 흐름

```
[STEP 1] 시황 판단 (공격/중립/방어)
[STEP 2] 대세섹터 확정 (고정 4대 + 당일 주도 섹터)
[STEP 3] 후보 풀 수집
         ├─ 단기 후보 (수급+기술) → 5~7
         ├─ 중기 후보 (펀더+수급 스코어) → 5~7
         └─ 장기 후보 (ROE·Moat·퀀트) → 5~8
[STEP 4] 눌림목 건강도 스코어링 (각 후보에 대해 0~100점)
[STEP 5] 대세(+)/대세(−) 태깅 → 6버킷 분류
[STEP 6] 버킷별 건강한 눌림 순 정렬 → 최종 리포트 출력
```

---

## STEP 1. 시황 판단

```bash
python 중기유망종목/scripts/naver_finance.py market
```
출력에서 다음 확정:
- **시장 국면**: 상승추세 초입 / 상승추세 지속 / 조정 중 / 하락추세
- **단기 매매 기조**: 공격 / 중립 / 방어
  - 공격: 외인+기관 쌍끌이 + 거래대금 20조↑
  - 중립: 수급 혼조
  - 방어: 외인+기관 쌍매도

> ⚠️ **방어 심화(3일 이상 지속) 시** → 발굴은 수행하되, 모든 종목에 `진입 보류` 플래그 부착

## STEP 3. 후보 풀 수집 (병렬 실행 가능)

### 3-A. 단기 후보 (5~7종목)
기준: `01_단기-유망종목-판별기준.md`

```bash
# 수급 스크리닝 (외국인·기관 순매수, 거래량 급증)
# 출력: 단기유망종목/.agent/scratch/stocks_data.txt (종목명+가격 포함)
python 단기유망종목/.agent/scratch/fetch_top_stocks.py
```

필터 순서:
1. 외국인 순매수 상위 10 ∩ 기관 순매수 상위 10
2. 거래량 전일比 2배 + 상승 마감
3. 당일 상승률 상위 테마 내 대장주
4. 이평선 5·20·60 **정배열**
5. RSI 과열 제외 (급등 3일 연속 ↑ 제외)
6. 공시 리스크 (5거래일 내 유증/CB) 제외

> **⚙️ 캐시 저장 (Claude가 수행)**
> 위 필터를 통과한 종목의 **6자리 코드만** 추출하여 아래 파일에 저장:
> ```
> workflows/.cache/YYYYMMDD_short.txt
> 예시: 005930 000660 042700 012450
> ```

---

### 3-B. 중기 후보 (5~7종목)
기준: `02_중기-유망종목-판별기준.md`

```bash
# 1단계: 대세·주도 섹터 업종번호로 종목 스크리닝
python 중기유망종목/scripts/naver_finance.py screen <업종번호>

# 2단계: 후보 종목별 종합 분석
python 중기유망종목/scripts/naver_finance.py all <종목코드>
python 중기유망종목/scripts/naver_finance.py investor <종목코드>
python 중기유망종목/scripts/naver_finance.py short <종목코드>
```

필터:
- 시총 500억↑, 커버리지 2개사↑, 목표주가 괴리율 +15%↑, 분기 영업흑자
- ROE 10%↑, 부채 150%↓, 유동 100%↑
- **스코어카드 Step 2(10점) + Step 3(10점) 합산 14점↑ 선정**

> **⚙️ 캐시 저장 (Claude가 수행)**
> 최종 선정 종목의 **6자리 코드만** 추출하여 저장:
> ```
> workflows/.cache/YYYYMMDD_medium.txt
> ```

### 3-C. 장기 후보 (5~8종목)

## STEP 4. 눌림목 건강도 스코어링

### 4-0. 스코어링 전 사전 확인 (Claude 수동 검증)

> ⚠️ 스크립트는 수급·가격 데이터만 체크한다. 아래 2가지는 Claude가 직접 확인한다.

**① 악재 공시 체크** (네이버 증권 → 각 종목 뉴스/공시 탭)
```
확인 항목 (하나라도 해당 시 → 부록 "탈락 목록"으로 이동):
  ❌ 최근 5거래일 내 유상증자·CB 발행 공시
  ❌ 실적 쇼크 / 가이던스 하향
  ❌ 횡령·배임·소송 등 지배구조 리스크
  ❌ 대형 계약 해지
  ❌ 섹터 전체 외국인·기관 동반 이탈 (3주 연속)
```

**② 시황 연동** (STEP 1 결과 적용)
```
STEP 1 시황 판단 결과를 여기서 반드시 반영:
  - "하락추세" 또는 "방어 심화(3일↑)" → 전 종목에 진입_보류 플래그 부착
    (스코어링은 수행하되, 리포트 모든 종목에 🚫 진입 보류 표시)
  - "방어(1~2일)" → 단기 후보 전체 진입_보류, 중기·장기는 정상 진행
  - "중립" or "공격" → 정상 진행
```

---

### 4-1. 스코어러 실행

모든 후보 풀을 합쳐 `pullback-scorer` 스킬 일괄 호출:

```bash
# 후보 코드를 공백 구분으로 모두 입력
python workflows/skills/pullback-scorer/pullback_score.py \
    $(cat workflows/.cache/YYYYMMDD_short.txt) \
    $(cat workflows/.cache/YYYYMMDD_medium.txt) \
    $(cat workflows/.cache/YYYYMMDD_long.txt) \
    --json > workflows/.cache/YYYYMMDD_scores.json
```

결과(JSON 배열)에 각 종목의 `total`(점수), `grade`(S/A/B/C/✕), `closest_ma`, 세부 배점이 담김.

### 4-2. 처리 원칙
- `eligible=false` 종목 → 자동 제외 (배제 조건 해당)
- `grade=✕` 종목 → 리포트 본문에 미포함 (부록 "탈락 목록"에만 기재)
- **4-0에서 악재 공시 해당 종목** → 점수 무관하게 부록으로 이동
- **4-0에서 진입_보류 플래그 종목** → 본문 포함하되 🚫 표시
- 기간별 본문 포함 등급:
  - **단기**: S/A만 → 타점이 수익의 전부, B 이하는 "대기 목록"으로
  - **중기**: S/A/B → 며칠 이른 진입 허용
  - **장기**: S/A/B/C → 종목 질이 타점보다 중요, C는 "분할 1차" 개념
- **C등급(단기) / C등급(중기)** 종목 → "대기 목록" 섹션에 기재

## STEP 5. 6버킷 태깅

각 종목에 대해:

**① 기간 분류** (STEP 3에서 어느 풀 출신인지로 결정)
- 단기 풀 출신 → `term=short`
- 중기 풀 출신 → `term=medium`
- 장기 풀 출신 → `term=long`

**② 대세 분류** (`04_대세섹터-판별기준.md §1` 적용)
- 종목의 업종이 대세섹터 리스트 포함 → `sector_status=대세(+)`
- 그 외 → `sector_status=대세(−)`

→ 6버킷:
```
short-major,    short-minor
medium-major,   medium-minor
long-major,     long-minor
```

## STEP 6. 최종 리포트 생성

템플릿: `06_최종-리포트-템플릿.md` 참조

출력 경로:
```
workflows/reports/YYYYMMDD_발굴리포트.md
```

### 리포트 구조 (요약)
1. **헤더**: 작성일, 시황, 대세섹터 리스트
2. **섹션 1 — 단기 유망 (단기×대세+ / 단기×대세−)** 건강도 순 정렬
3. **섹션 2 — 중기 유망 (중기×대세+ / 중기×대세−)** 건강도 순 정렬
4. **섹션 3 — 장기 유망 (장기×대세+ / 장기×대세−)** 건강도 순 정렬
5. **대기 목록**: 타점 미도달 종목 (단기 B/C, 중기/장기 C 중 관망 대기)
6. **부록**: 탈락 목록, 배제 사유

각 종목마다:
- 눌림목 점수(등급) / 조정%·일수 / 주 타점 / 수급 상태
- 현재가·목표가·손절가
- 핵심 투자 근거 1~2줄

---

## 운영 원칙

1. **데이터 출처 네이버 증권 단독** — DART·증권사·타 포털 금지
2. **발굴 전용** — 보유 포트폴리오 무시, 신규 후보만 제시
3. **방어 시황 시** — 발굴은 수행하되 전 종목에 `진입 보류` 플래그
4. **중복 종목 처리** — 단/중/장 풀에 동시 등장 시 **점수가 가장 높은 기간**에만 배치
5. **실시간 시세** — 리포트 확정 직전 `naver_finance.py stock <코드>` 재호출하여 현재가 갱신
6. **리포트 푸시** — `"push 하자"` 요청 시에만 별도 push 워크플로우 실행 (이 워크플로우는 로컬 저장까지)
7. **미사용 스크립트** — `fetch_index.py`(naver_finance.py market으로 대체), `fetch_details.py`(테스트용) 는 이 워크플로우에서 호출하지 않음

---

## 성공 기준

- 리포트에 **단·중·장 섹션 모두 존재** (각 섹션 최소 1종목, 없으면 "해당 없음" 명시)
- **건강도 점수 DESC + 20일선 거리 ASC + 시총 DESC** 3중 정렬 준수
- **모든 종목에 현재가·목표가·손절가 명시**
- **배제 종목은 본문 제외, 부록에만 기재**
