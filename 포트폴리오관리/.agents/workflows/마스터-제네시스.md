# 마스터 제네시스 (Master Genesis)

## 실행 트리거
사용자가 **"제네시스 하자"** 입력 시 이 워크플로우를 즉시 가동한다.

---

## 1단계: 한국 날짜/요일 확인

```bash
# 한국 시간(KST) 기준 오늘 날짜와 요일 확인
python -c "
from datetime import datetime
import pytz
kst = pytz.timezone('Asia/Seoul')
now = datetime.now(kst)
print(f'날짜: {now.strftime(\"%Y-%m-%d\")}')
print(f'요일: {now.strftime(\"%A\")}')
print(f'일(day): {now.day}')
print(f'요일번호: {now.weekday()}')  # 0=월, 4=금, 5=토, 6=일
"
```

확인 후 아래 실행 매트릭스에 따라 워크플로우를 순서대로 실행한다.

---

## 2단계: 실행 매트릭스

### 요일별 실행 워크플로우

| 조건 | 실행 워크플로우 |
|------|--------------|
| 월~목 | ① 시황분석 → ② 단기유망종목 → ③ 포트폴리오 리뷰 |
| 금요일 | ① 시황분석 → ② 단기유망종목 → ③ 중기유망종목 → ④ 포트폴리오 리뷰 |
| 매달 1일 (요일 무관) | 위 요일 기준에 추가로 ⑤ 장기유망종목 실행 |
| 토/일 | ③ 포트폴리오 리뷰만 실행 (시장 휴장) |

### 우선순위 판단 순서
```
1. 오늘이 매달 1일인가?
   → YES: 해당 요일 기준 워크플로우 + 장기유망종목 추가
   → NO: 다음 단계

2. 오늘이 금요일인가?
   → YES: 시황 + 단기 + 중기 + 포트리뷰
   → NO: 다음 단계

3. 오늘이 월~목인가?
   → YES: 시황 + 단기 + 포트리뷰
   → NO (토/일): 포트리뷰만
```

---

## 3단계: 워크플로우 순차 실행

### ① 시황분석 (월~금 매일)
- 워크플로우: `시황분석/workflows/제네시스.md`
- 실행 후 결과물 저장 경로 확인:
  `시황분석/content/market-analysis/YYYY-MM-DD-market-analysis-genesis.mdx`
- **포트폴리오 리뷰를 위해 아래 항목을 반드시 메모한다:**
  - 오늘 시장 국면 (상승추세/조정중/하락추세)
  - 주도 섹터 및 약세 섹터
  - 핵심 리스크 요인
  - 오늘의 매매 전략 (Do/Don't)

### ② 단기유망종목 (월~금 매일)
- 워크플로우: `단기유망종목/.agent/workflows/제네시스-단기유망종목.md`
- 실행 후 결과물 저장 경로 확인:
  `단기유망종목/YYYYMMDD_단기유망종목_발굴리포트.md`
- **포트폴리오 리뷰를 위해 아래 항목을 반드시 메모한다:**
  - 오늘 추천 종목 목록 (종목명, 현재가, 목표가, 손절가)
  - 시장 판단 (공격/중립/방어)

### ③ 중기유망종목 (금요일만)
- 워크플로우: `중기유망종목/.agents/workflows/제네시스-중기유망.md`
- 실행 후 결과물 저장 경로 확인:
  `중기유망종목/content/picks/YYYYMMDD-genesis-report.mdx`
- **포트폴리오 리뷰를 위해 아래 항목을 반드시 메모한다:**
  - 현재 추천 중기 종목 목록 (종목명, 목표가, 손절가, R:R)
  - 다음주 매수 우선순위

### ③-1. 대세 섹터 주간 모니터링 (금요일만)
중기유망종목 분석 완료 후 아래를 체크한다.

```
포트폴리오관리/포트폴리오-대장.md의 "현재 지정 대세 섹터" 기준으로:

[편입 후보 체크]
- 이번 주 외국인·기관 동반 순매수가 4주째 지속된 새 섹터가 있는가?
- 글로벌 CAPEX 발표·정책 입법 등 구조 트리거가 새로 공식화된 섹터가 있는가?
→ 해당 시: 🔍 편입후보 메모 남기기

[강등 후보 체크]
- 현재 대세 섹터 중 외국인·기관 동반 순매도가 4주째 지속된 곳이 있는가?
- 글로벌 트리거 훼손(CAPEX 축소·정책 후퇴) 공식 발표가 있었는가?
- 대표 기업 2개 이상 가이던스 하향·어닝쇼크가 발생했는가?
→ 해당 시: ⚠️ 강등후보 메모 남기기

→ 신호가 없으면: "대세 섹터 변동 없음 (이번 주 신호 없음)" 한 줄로 기재
→ 확정(편입·강등)은 하지 않는다. 다음 매달 1일 장기 제네시스에서 최종 결정.
```

### ④ 장기유망종목 (매달 1일만)
- 워크플로우: `장기유망종목/.agents/workflows/genesis-long-term.md`
- 실행 후 결과물 저장 경로 확인:
  `장기유망종목/reports/genesis_long_term_report_YYYYMMDD.md`
- **이 워크플로우에는 7단계(대세 섹터 월간 공식 리뷰)가 포함된다. 종목 발굴 완료 후 자동으로 이어서 실행.**
- **포트폴리오 리뷰를 위해 아래 항목을 반드시 메모한다:**
  - 현재 추천 장기 종목 목록
  - 각 종목 thesis 유효 여부
  - 대세 섹터 변경 사항 (편입·강등 확정 내용)

## 4단계: 포트폴리오 리뷰 실행

위 워크플로우들이 모두 완료된 후, **포트폴리오 대장을 먼저 읽고** 안내 메시지를 출력한다.

### 4-0. 포트폴리오 대장 읽기 (필수)
```
읽을 파일: 포트폴리오관리/포트폴리오-대장.md
확인 항목:
  [기본 정보]
  - 현재 등록된 보유 종목 목록
  - 각 종목의 분류(중기/장기), 섹터구분(일반/대세)
  - 목표가, 1차/2차 손절가, 현재 상태(익절 진행 단계 등)

  [예산 현황]
  - 이달 기본 예산 잔여액 (220만원 중 미투입분)
  - 추가 재원 잔액 (익절/손절 회수금 중 미재투자분)
  - 손절 발생일 (3거래일 쿨다운 계산용)

→ 위 정보를 포트폴리오 리뷰 전체의 기준값으로 사용한다
```

### 4-0-1. 오늘 시황 판단 정리 (포트리뷰 전달용)
```
앞서 실행된 단기유망종목 리포트에서 확인:
  - 시장 판단: [공격 / 중립 / 방어]  ← 투입 강도 결정 기준
  - 주도 섹터: [섹터명]
  - 오늘의 Do/Don't

→ 이 판단이 아래 포트리뷰의 투입 강도 및 추가 재원 운용 기준이 된다
   공격: 기본예산 적극 투입 + 추가재원 재투자 가능
   중립: 기본예산 50% 이하 투입 + 추가재원 50%만 재투자
   방어: 기본예산 투입 전면 보류 + 추가재원 전액 현금 보유
```

| 사용자 입력 | 처리 |

| "오늘 리뷰 없음" | 워크플로우 종료 |

### 4-1. playwright MCP 자동 크롤링 (캡처 미제공 시)

사용자가 "playwright 크롤링 해줘" 입력 시 아래 순서로 실행한다.

#### Step 1. 토스증권 접속 및 로그인 확인
```
browser_navigate → 토스증권 웹 (https://securities.toss.im)
browser_snapshot → 현재 화면 상태 확인
- 이미 로그인된 세션이면 바로 진행
- 로그인 페이지가 뜨면: "토스증권 로그인이 필요합니다. 브라우저에서 로그인 후 '계속'이라고 입력해주세요." 안내
```

#### Step 2. 계좌 전체 수집
```
[수집 항목]
- 총 평가금액
- 총 투자원금
- 총 손익액 / 총 수익률
- 예수금 (국내 / 해외)
```

#### Step 3. 국내주식 보유현황 수집
```
국내주식 탭으로 이동 후 browser_snapshot

[수집 항목 - 보유 종목별]
- 종목명
- 보유수량
- 평균단가
- 현재가
- 평가금액
- 손익금액 / 수익률
```

#### Step 4. 해외주식 보유현황 수집
```
해외주식 탭으로 이동 후 browser_snapshot

[수집 항목 - 보유 종목별]
- 종목명 (티커)
- 보유수량
- 평균단가 (원화 환산 포함)
- 현재가
- 평가금액 (원화 환산 포함)
- 손익금액 / 수익률
```

#### Step 5. 당일 주문내역 수집
```
주문내역 / 거래내역 탭으로 이동 후 browser_snapshot

[수집 항목 - 당일 체결 건]
- 매수/매도 구분
- 종목명
- 체결수량
- 체결가
- 체결금액
- 체결시간
```

#### Step 6. 크롤링 데이터 정리 및 리뷰 실행
```
수집된 데이터를 아래 형태로 구조화:

=== playwright 크롤링 결과 (YYYY-MM-DD) ===
[계좌 전체]
총 평가금액: XXX원 | 총 손익: XXX원 (XX%)
예수금(국내): XXX원 | 예수금(해외): XXX원

[국내주식 보유현황]
종목명 | 수량 | 평균단가 | 현재가 | 평가금액 | 수익률
...

[해외주식 보유현황]
종목명 | 수량 | 평균단가(원화) | 현재가(원화) | 평가금액 | 수익률
...

[당일 매수내역]
종목명 | 수량 | 체결가 | 금액 | 시간
...

[당일 매도내역]
종목명 | 수량 | 체결가 | 금액 | 시간
...
========================================

→ 이 데이터를 INPUT으로 `포트폴리오관리/.agents/workflows/데일리-포트폴리오-리뷰.md` 실행
```

> ⚠️ 크롤링 중 페이지 구조가 달라 데이터를 찾지 못하면, 해당 항목은 "수집 실패 - 수동 입력 필요"로 표기하고 나머지 데이터로 리뷰를 진행한다.

## 주의사항
- 각 워크플로우는 반드시 **순서대로** 실행한다 (시황 → 단기 → 중기 → 장기 → 포트리뷰)
- 포트폴리오 리뷰는 앞선 워크플로우 결과물을 INPUT으로 사용하므로 마지막에 실행
- Push는 사용자가 각 폴더에서 직접 실행 (이 워크플로우에서 push 금지)
- 주말(토/일)에는 시황분석, 종목 워크플로우 실행 안 함 (시장 휴장)
