# 마스터 제네시스 (Master Genesis)

## 실행 트리거
사용자가 **"제네시스 하자"** 입력 시 이 워크플로우를 즉시 가동한다.

---

## 1단계: 한국 날짜/요일 확인

```bash
# 한국 시간(KST) 기준 오늘 날짜와 요일 확인
python -c "
from datetime import datetime
import pytz
kst = pytz.timezone('Asia/Seoul')
now = datetime.now(kst)
print(f'날짜: {now.strftime(\"%Y-%m-%d\")}')
print(f'요일: {now.strftime(\"%A\")}')
print(f'일(day): {now.day}')
print(f'요일번호: {now.weekday()}')  # 0=월, 4=금, 5=토, 6=일
"
```

확인 후 아래 실행 매트릭스에 따라 워크플로우를 순서대로 실행한다.

---

## 2단계: 실행 매트릭스

### 요일별 실행 워크플로우

| 조건 | 실행 워크플로우 |
|------|--------------|
| 월~목 | ① 시황분석 → ② 단기유망종목 → ③ 포트폴리오 리뷰 |
| 금요일 | ① 시황분석 → ② 단기유망종목 → ③ 중기유망종목 → ④ 포트폴리오 리뷰 |
| 매달 1일 (요일 무관) | 위 요일 기준에 추가로 ⑤ 장기유망종목 실행 |
| 토/일 | ③ 포트폴리오 리뷰만 실행 (시장 휴장) |

### 우선순위 판단 순서
```
1. 오늘이 매달 1일인가?
   → YES: 해당 요일 기준 워크플로우 + 장기유망종목 추가
   → NO: 다음 단계

2. 오늘이 금요일인가?
   → YES: 시황 + 단기 + 중기 + 포트리뷰
   → NO: 다음 단계

3. 오늘이 월~목인가?
   → YES: 시황 + 단기 + 포트리뷰
   → NO (토/일): 포트리뷰만
```

## 3단계: 워크플로우 순차 실행

### ① 시황분석 (월~금 매일)
- 워크플로우: `시황분석/workflows/제네시스.md`
- 실행 후 결과물 저장 경로 확인:
  `시황분석/content/market-analysis/YYYY-MM-DD-market-analysis-genesis.mdx`
- **포트폴리오 리뷰를 위해 아래 항목을 반드시 메모한다:**
  - 오늘 시장 국면 (상승추세/조정중/하락추세)
  - 주도 섹터 및 약세 섹터
  - 핵심 리스크 요인
  - 오늘의 매매 전략 (Do/Don't)

### ② 단기유망종목 (월~금 매일)
- 워크플로우: `단기유망종목/.agent/workflows/제네시스-단기유망종목.md`
- 실행 후 결과물 저장 경로 확인:
  `단기유망종목/` 하위 최신 리포트
- **포트폴리오 리뷰를 위해 아래 항목을 반드시 메모한다:**
  - 오늘 추천 종목 목록 (종목명, 현재가, 목표가, 손절가)
  - 시장 판단 (공격/중립/방어)

### ③ 중기유망종목 (금요일만)
- 워크플로우: `중기유망종목/.agents/workflows/제네시스-중기유망.md`
- 실행 후 결과물 저장 경로 확인:
  `중기유망종목/content/picks/` 하위 최신 리포트
- **포트폴리오 리뷰를 위해 아래 항목을 반드시 메모한다:**
  - 현재 추천 중기 종목 목록 (종목명, 목표가, 손절가, R:R)
  - 다음주 매수 우선순위

### ③-1. 대세 섹터 주간 모니터링 (금요일만)
중기유망종목 분석 완료 후 아래를 체크한다.

```
포트폴리오관리/포트폴리오-대장.md의 "현재 지정 대세 섹터" 기준으로:

[편입 후보 체크]
- 이번 주 외국인·기관 동반 순매수가 4주째 지속된 새 섹터가 있는가?
- 글로벌 CAPEX 발표·정책 입법 등 구조 트리거가 새로 공식화된 섹터가 있는가?
→ 해당 시: 🔍 편입후보 메모 남기기

[강등 후보 체크]
- 현재 대세 섹터 중 외국인·기관 동반 순매도가 4주째 지속된 곳이 있는가?
- 글로벌 트리거 훼손(CAPEX 축소·정책 후퇴) 공식 발표가 있었는가?
- 대표 기업 2개 이상 가이던스 하향·어닝쇼크가 발생했는가?
→ 해당 시: ⚠️ 강등후보 메모 남기기

→ 신호가 없으면: "대세 섹터 변동 없음 (이번 주 신호 없음)" 한 줄로 기재
→ 확정(편입·강등)은 하지 않는다. 다음 매달 1일 장기 제네시스에서 최종 결정.
```

## 4단계: 포트폴리오 리뷰 실행

위 워크플로우들이 모두 완료된 후, **포트폴리오 대장을 먼저 읽고** 안내 메시지를 출력한다.

### 4-0. 포트폴리오 대장 읽기 (필수)
```

  - 현재 등록된 보유 종목 목록

  - 목표가, 1차/2차 손절가, 현재 상태(익절 진행 단계 등)

  [예산 현황]
  - 이달 기본 예산 잔여액 (220만원 중 미투입분)

  - 손절 발생일 (3거래일 쿨다운 계산용)

→ 위 정보를 포트폴리오 리뷰 전체의 기준값으로 사용한다
```

### 업로드 절차

각 파일에 대해 아래 순서로 실행한다:

**1. Python으로 base64 인코딩 (홈 디렉토리에 임시 저장)**

날짜 자동 계산 및 요일 조건 판단을 포함한다. 파일이 존재하지 않으면 건너뛴다(토/일·리뷰 없음 시 안전 처리).

```python
import base64, os
from datetime import datetime
import pytz

kst = pytz.timezone('Asia/Seoul')
now = datetime.now(kst)
ymd = now.strftime('%Y-%m-%d')      # 2026-04-21
ymd8 = now.strftime('%Y%m%d')       # 20260421
is_friday = now.weekday() == 4
is_first = now.day == 1
is_weekday = now.weekday() < 5

home = os.path.expandvars('%USERPROFILE%')
base = os.path.join(home, 'OneDrive', '바탕 화면', '포트폴리오분석')

# 기본 4개 (평일만 존재, 주말엔 시황/단기/리뷰 없음)
candidates = {
    'gdrive_b64_1.txt': os.path.join(base, '포트폴리오관리', '포트폴리오-대장.md'),
}
if is_weekday:
    candidates['gdrive_b64_2.txt'] = os.path.join(base, '시황분석', 'content', 'market-analysis', f'{ymd}-market-analysis-genesis.mdx')
    candidates['gdrive_b64_3.txt'] = os.path.join(base, '단기유망종목', f'{ymd8}_단기유망종목_발굴리포트.md')
    candidates['gdrive_b64_4.txt'] = os.path.join(base, '포트폴리오관리', 'reviews', f'{ymd}-review.md')
if is_friday:
    candidates['gdrive_b64_5.txt'] = os.path.join(base, '중기유망종목', 'content', 'picks', f'{ymd8}-genesis-report.mdx')
if is_first:
    candidates['gdrive_b64_6.txt'] = os.path.join(base, '장기유망종목', 'reports', f'genesis_long_term_report_{ymd8}.md')

for out, path in candidates.items():
    if not os.path.exists(path):
        print(f'SKIP (없음): {path}')
        continue
    with open(path, 'rb') as f:
        data = f.read()
    b64 = base64.b64encode(data).decode('ascii')
    dest = os.path.join(home, out)
    with open(dest, 'w') as f:
        f.write(b64)
    print(out + ' OK len=' + str(len(b64)))
```

**2. 생성된 임시 파일만 Read 툴로 읽는다**

위 스크립트 출력에서 "OK"로 표시된 파일만 읽는다. "SKIP"된 파일은 건너뛴다.

```
Read: C:/Users/hwang/gdrive_b64_1.txt  ← 항상
Read: C:/Users/hwang/gdrive_b64_2.txt  ← 평일만
Read: C:/Users/hwang/gdrive_b64_3.txt  ← 평일만
Read: C:/Users/hwang/gdrive_b64_4.txt  ← 평일 + 리뷰 있을 때만
Read: C:/Users/hwang/gdrive_b64_5.txt  ← 금요일만
Read: C:/Users/hwang/gdrive_b64_6.txt  ← 매달 1일만
```

**3. mcp__claude_ai_Google_Drive__create_file 로 동시 업로드**

존재하는 파일만 업로드한다 (SKIP된 파일 제외).

```
mimeType: text/plain  (자동으로 Google Docs로 변환됨)
content: 위에서 읽은 base64 문자열 (줄바꿈 없는 단일 문자열 그대로)
```

**4. 업로드 완료 후 임시 파일 삭제**

```python
import os
home = os.path.expandvars('%USERPROFILE%')
for i in range(1, 7):
    path = os.path.join(home, f'gdrive_b64_{i}.txt')
    if os.path.exists(path):
        os.remove(path)
print('임시 파일 삭제 완료')
```

## 주의사항
- 각 워크플로우는 반드시 **순서대로** 실행한다 (시황 → 단기 → 중기 → 장기 → 포트리뷰)
- 포트폴리오 리뷰는 앞선 워크플로우 결과물을 INPUT으로 사용하므로 마지막에 실행
- Push는 사용자가 각 폴더에서 직접 실행 (이 워크플로우에서 push 금지)
- 주말(토/일)에는 시황분석, 종목 워크플로우 실행 안 함 (시장 휴장)
- Google Drive 업로드는 6단계에서 자동 실행 (별도 요청 불필요)
