# GitHub Push 워크플로우 (유망종목-장기)

이 워크플로우는 생성된 장기 유망 종목 보고서를 지정된 GitHub 저장소의 '유망종목(장기)' 섹션으로 배포하는 절차를 정의합니다.

## 1. 배포 대상 정보
- **저장소 URL**: `https://github.com/pwman111-debuge/stockanalysis`
- **대상 디렉토리**: `유망종목-장기` (웹사이트의 '유망종목(장기)' 메뉴와 연동)

## 2. 배포 트리거
- 황원장님이 "push 하자" 또는 "GitHub에 배포해줘"라고 명령할 때 실행합니다.

## 3. 배포 절차

### 단계 1: 배포 파일 확인
- `reports/` 폴더 내에 새로 생성되거나 수정된 `.md` 또는 `.mdx` 보고서 파일을 확인합니다.

### 단계 2: Git 상태 점검 및 스테이징
- 로컬 변경 사항을 확인합니다.
- 배포할 보고서 파일을 스테이징합니다.
  ```powershell
  git add reports/*.md
  ```

### 단계 3: 커밋 실행
- 보고서 생성 날짜와 종목명을 포함하여 의미 있는 커밋 메시지를 작성합니다.
- 예: `feat: 유망종목(장기) 보고서 업데이트 (2026-04-09)`

### 단계 4: 원격 저장소 푸시 // turbo
- 지정된 저장소로 푸시를 수행합니다.
  ```powershell
  git push origin main
  ```

## 4. 주의사항
- 배포 전 반드시 보고서의 데이터 무결성(검증 단계)이 완료되었는지 확인합니다.
- 원격 저장소와의 충돌이 있을 경우, `pull`을 먼저 수행하여 병합 후 푸시합니다.
